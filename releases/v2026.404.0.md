# v2026.404.0

> Released: 2026-04-04

## Highlights

- **Desktop app and packaged runtime polish** — Paperclip CN now ships a fuller Electron desktop shell with packaged runtime optimization, embedded Postgres preflight repair, and native titlebar navigation/divider polish so desktop installs behave more like a first-class app. ([#21](https://github.com/penclipai/paperclip-cn/pull/21), [#22](https://github.com/penclipai/paperclip-cn/pull/22), [#23](https://github.com/penclipai/paperclip-cn/pull/23), [#28](https://github.com/penclipai/paperclip-cn/pull/28), @penclipai)
- **Execution workspaces** — Full workspace lifecycle management landed for agent runs: workspace-aware routine runs, execution workspace detail pages with linked issues, runtime controls, close readiness checks, and follow-up issue workspace inheritance. ([#2074](https://github.com/paperclipai/paperclip/pull/2074), [#2203](https://github.com/paperclipai/paperclip/pull/2203))
- **Inbox, feedback, and document revisions** — The Mine inbox overhaul, thumbs-up/down feedback flow, and issue document revision restore flow now work together to give operators a faster review loop. ([#2072](https://github.com/paperclipai/paperclip/pull/2072), [#2529](https://github.com/paperclipai/paperclip/pull/2529), [#2317](https://github.com/paperclipai/paperclip/pull/2317))
- **Local adapter expansion** — CodeBuddy and Qwen local adapters join the supported local stack, alongside Gemini validation hardening and GPT-5.4/xhigh effort options from upstream. ([#16](https://github.com/penclipai/paperclip-cn/pull/16), [#2430](https://github.com/paperclipai/paperclip/pull/2430), [#112](https://github.com/paperclipai/paperclip/pull/112), @penclipai, @bittoby, @kevmok)
- **Chinese localization and locale cleanup** — Multiple upstream syncs plus runtime locale materialization cleanup keep onboarding, feedback, agent instructions, and operator surfaces aligned in zh-CN. ([#13](https://github.com/penclipai/paperclip-cn/pull/13), [#14](https://github.com/penclipai/paperclip-cn/pull/14), [#17](https://github.com/penclipai/paperclip-cn/pull/17), [#20](https://github.com/penclipai/paperclip-cn/pull/20), [#24](https://github.com/penclipai/paperclip-cn/pull/24), [#26](https://github.com/penclipai/paperclip-cn/pull/26), @penclipai)

## Improvements

- **Telemetry and routines** — App-side telemetry sender, plugin telemetry bridge support, backend-aligned event payloads, and workspace-aware routines all landed in the same release window. ([#2414](https://github.com/paperclipai/paperclip/pull/2414), [#2527](https://github.com/paperclipai/paperclip/pull/2527), @aronprins)
- **GitHub Enterprise imports** — Skill and company imports now accept GitHub Enterprise URLs with hardened detection and shared GitHub helpers. ([#2449](https://github.com/paperclipai/paperclip/pull/2449), @statxc)
- **Operator and editor polish** — Instruction editors now use monospace text, org chart cards render agent capabilities, markdown links are styled as links, and onboarding reruns preserve existing config. ([#2349](https://github.com/paperclipai/paperclip/pull/2349), [#2620](https://github.com/paperclipai/paperclip/pull/2620), [#2071](https://github.com/paperclipai/paperclip/pull/2071))
- **Commit metrics and desktop ergonomics** — Paperclip commit metric exports, instructions layout polish, and titlebar navigation updates improve both maintainer workflows and packaged desktop navigation. ([#23](https://github.com/penclipai/paperclip-cn/pull/23), [#28](https://github.com/penclipai/paperclip-cn/pull/28), @penclipai)

## Fixes

- **Windows and local runtime recovery** — Fixed Windows startup and shutdown recovery, local skill injection fallback, shell fallback behavior, and issue status filtering so local environments are less brittle. ([#13](https://github.com/penclipai/paperclip-cn/pull/13), [#27](https://github.com/penclipai/paperclip-cn/pull/27), [#891](https://github.com/paperclipai/paperclip/pull/891), @penclipai)
- **Localization regressions** — Remaining zh-CN gaps in the inbox timeline, feedback run labels, agent instructions, and costs UI were cleaned up across the sync series. ([#17](https://github.com/penclipai/paperclip-cn/pull/17), [#20](https://github.com/penclipai/paperclip-cn/pull/20), [#24](https://github.com/penclipai/paperclip-cn/pull/24), [#26](https://github.com/penclipai/paperclip-cn/pull/26), @penclipai)
- **Docker and environment handling** — Fixed Plain-to-Secret env var data loss, preserved adapter-agnostic keys during adapter switches, improved Docker image/runtime layout, and hardened volume mount behavior. ([#2327](https://github.com/paperclipai/paperclip/pull/2327), [#2407](https://github.com/paperclipai/paperclip/pull/2407), [#1923](https://github.com/paperclipai/paperclip/pull/1923), @radiusred)
- **Inbox, worktree, and runtime stability** — Self-touched issues no longer sink in the inbox, worktree creation is more robust, Codex spawn ENOENT handling is fixed, and timer or heartbeat wakes preserve session continuity. ([#2144](https://github.com/paperclipai/paperclip/pull/2144), [#2463](https://github.com/paperclipai/paperclip/pull/2463), [#2048](https://github.com/paperclipai/paperclip/pull/2048), [#2065](https://github.com/paperclipai/paperclip/pull/2065), @remdev, @edimuj)

## Upgrade Guide

- Three new database migrations (`0046`-`0048`) will run automatically on startup. These add routines, feedback, and document revision storage, plus related schema support. The migrations are additive.
- If you distribute the packaged desktop build on Windows, startup now performs embedded Postgres preflight repair automatically; no extra manual repair step is required.
- Existing `penclip` CLI installs do not need a command rename or config migration for this release.
