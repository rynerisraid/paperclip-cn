# v2026.406.0

> Released: 2026-04-06

## Highlights

- **External adapter plugins** — Paperclip now supports external adapter plugins with a declarative config-schema API, dynamic UI parsers, built-in adapter overrides, and better adapter manager visibility into source, version, override state, and reinstall actions. ([#2218](https://github.com/paperclipai/paperclip/pull/2218), @HenkDz)
- **Hermes and adapter control upgrades** — Hermes local gains session management plus provider/model visibility in run details, and built-in adapters can now be paused or resumed through server-side override controls. ([#2218](https://github.com/paperclipai/paperclip/pull/2218), @HenkDz)
- **Operator workflow polish** — The routines list, recent runs, issue comment wake prompts, slash-command autocomplete, and search on the issues page all received usability improvements for faster day-to-day operation.
- **Mobile and settings UX cleanup** — Toggle switches are now unified into a responsive component, the close-workspace modal behaves better on mobile, mentions no longer misplace or blur-save unexpectedly, and instance settings now expose a sign-out button.

## Improvements

- **Worktree provisioning reliability** — Issue worktrees now provision local `node_modules`, harden dependency hydration, and reconcile workspace runtime state more reliably.
- **Windows developer stability** — Windows full-suite Vitest runs, worktree provisioning tests, embedded Postgres lifecycle cleanup, and stale company skill selection handling were all tightened in the CN branch. ([#30](https://github.com/penclipai/paperclip-cn/pull/30), @penclipai)
- **UI fit and finish** — Transcript views gained a collapsible `system_group` block, recent runs and routines render more cleanly, and toggle-heavy screens now stay consistent across desktop and mobile layouts.

## Fixes

- **Mobile layout bugs** — Fixed unified toggle behavior on mobile, close-workspace modal overflow, mention menu placement, and other responsive edge cases. ([#2749](https://github.com/paperclipai/paperclip/pull/2749))
- **Workspace lifecycle bugs** — Closed isolated workspaces are now guarded correctly, runtime state reconciliation is more accurate, and workspace dependency setup is less fragile.
- **Adapter UI/parser bugs** — External adapter parser overrides now stay fresh, builtin parser overrides no longer leak UI state, and `overridePaused` handling is correctly narrowed to booleans. ([#2792](https://github.com/paperclipai/paperclip/pull/2792), @HenkDz)
- **Inline editing polish** — Pasted markdown is autoformatted in the inline editor, mention selection no longer triggers an accidental blur-save, and comment wake prompts are trimmed to the right scope. ([#2673](https://github.com/paperclipai/paperclip/pull/2673), @devinfoley)

## Upgrade Guide

- No new database migrations are introduced in this release.
- Existing `penclip` installs can upgrade in place; there is no CLI rename or config migration needed.
- If you maintain custom adapters or adapter plugins, this release is the first one where external adapter overrides and declarative config-schema support are part of the public product surface, so it is a good time to verify your adapter UI against the new override-aware manager.
