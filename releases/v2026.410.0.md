# v2026.410.0

> Released: 2026-04-10

## Highlights

- **Issue workflow overhaul** — Issues now support explicit review and approval execution stages, issue-linked board approvals, chat-style comment threads, richer activity and run presentation, and faster detail-page updates across the main operator workflow. ([#39](https://github.com/penclipai/paperclip-cn/pull/39), @penclipai)
- **Inbox and routines usability upgrades** — Inbox views now support parent-child nesting, better keyboard navigation, reusable column controls, and dismissal persistence, while routines gained draft defaults, run-time overrides, variable placeholders, and a clearer recent-runs flow. ([#39](https://github.com/penclipai/paperclip-cn/pull/39), @penclipai)
- **Desktop and release confidence improvements** — Desktop startup was hardened against stale smoke config and temp-home drift, acceptance coverage expanded, and the release smoke lane now tolerates the fork’s zh-CN-first auth and onboarding UI so published canaries can complete their full browser validation. ([#38](https://github.com/penclipai/paperclip-cn/pull/38), [#40](https://github.com/penclipai/paperclip-cn/pull/40), @penclipai)

## Improvements

- **CN-localized operator surface** — The upstream sync preserved the fork’s zh-CN locale defaults while restoring visible localization coverage across issues, inbox, routines, instance settings, and related high-traffic UI shells. ([#39](https://github.com/penclipai/paperclip-cn/pull/39), @penclipai)
- **Worktree and local-dev reliability** — Root dev commands now resolve `tsx` through a repo-local launcher instead of a fragile nested install path, and worktree reseed behavior now fails loudly when an explicit embedded-postgres source is missing instead of silently degrading into an empty reset. ([#39](https://github.com/penclipai/paperclip-cn/pull/39), @penclipai)
- **Execution and heartbeat stability** — The release includes fixes for signoff-stage transitions, non-participant stage rejection, heartbeat child PID persistence, comment wake retry paths, and multiple issue-chat rendering edge cases introduced during the upstream workflow changes.

## Fixes

- **Localized release smoke auth flow** — The authenticated Docker onboarding smoke test now accepts both English and zh-CN labels for sign-in and onboarding, preventing false release failures after UI localization. ([#40](https://github.com/penclipai/paperclip-cn/pull/40), @penclipai)
- **Issue detail regressions after upstream sync** — Fixed lost or hardcoded English UI strings, stabilized issue properties and document views, restored language switcher visibility in the shared layout, and realigned affected unit/e2e coverage with the current localized UI. ([#39](https://github.com/penclipai/paperclip-cn/pull/39), @penclipai)
- **Desktop startup path handling** — Fixed stale temp-home path handling across desktop/server/db startup so acceptance and local packaged runs do not inherit broken smoke configuration state. ([#38](https://github.com/penclipai/paperclip-cn/pull/38), @penclipai)

## Upgrade Guide

- This release includes new database migrations covering issue execution policy state, issue relations, inbox dismissals, heartbeat run metadata, and draft routine support. Let normal startup apply migrations automatically or run `pnpm db:migrate` before production rollout if you manage upgrades manually.
- Existing `penclip` installs can upgrade in place; there is no package rename or new auth bootstrap step required for the stable release.
- If you rely on the desktop packaging or release smoke flow, note that this release hardens startup and acceptance behavior without changing the public CLI install path.
