# v2026.411.0

> Released: 2026-04-11

## Highlights

- **Tiered backup retention and compressed backups** — Instance settings now expose daily, weekly, and monthly retention controls, and automatic database backups are stored as gzip-compressed archives instead of leaving large raw SQL dumps behind. ([#44](https://github.com/penclipai/paperclip-cn/pull/44), @aronprins, @penclipai)
- **Safer company-scoped operator actions** — Approval, activity, import, and related operator routes now enforce company access more strictly so cross-company actions are rejected earlier and more consistently. ([#44](https://github.com/penclipai/paperclip-cn/pull/44), @aronprins, @penclipai)
- **Latest document revision stays visible in issue detail** — Issue document panels now correctly treat the newest fetched revision as current even when summary data is stale, so reviewers can return from historical preview without getting stuck on an older revision. ([#44](https://github.com/penclipai/paperclip-cn/pull/44), @aronprins, @penclipai)

## Improvements

- **CN fork settings stayed aligned during the upstream sync** — The upstream merge kept Paperclip CN’s runtime default language control while adopting the new backup-retention controls, with matching `en` and `zh-CN` UI copy for the updated settings surface. ([#44](https://github.com/penclipai/paperclip-cn/pull/44), @penclipai)
- **Local dev boot is more reliable** — The CLI now prepares required workspace build artifacts before starting the dev server, reducing startup failures in plugin-aware dev sessions. ([#44](https://github.com/penclipai/paperclip-cn/pull/44), @aronprins, @penclipai)
- **Claude instruction bundles resolve sibling files more predictably** — Relative references such as `./HEARTBEAT.md`, `./SOUL.md`, and `./TOOLS.md` are now anchored to the instructions file directory instead of drifting to a parent folder. ([#44](https://github.com/penclipai/paperclip-cn/pull/44), @aronprins, @penclipai)

## Fixes

- **Backup cleanup leaves no orphaned files** — Failed backup/compression runs now clean up partial `.sql` or `.sql.gz` artifacts instead of leaving broken files behind. ([#44](https://github.com/penclipai/paperclip-cn/pull/44), @aronprins, @penclipai)
- **Approval retry coverage stayed intact through the sync** — The sync preserved the fork’s requester-wakeup locale behavior while taking upstream company-scope approval guards and related test coverage. ([#44](https://github.com/penclipai/paperclip-cn/pull/44), @penclipai)
- **Release and smoke validation stayed aligned with current master** — The post-`v2026.410.0` maintenance pass kept the release surface leaner and the latest smoke checks tolerant of the current master output shape instead of failing on duplicated summary text. ([#43](https://github.com/penclipai/paperclip-cn/pull/43), @penclipai)

## Upgrade Guide

- Existing `penclip` installs can upgrade in place; there is no package rename or new onboarding step for this stable release.
- Automatic backup retention now defaults to a tiered policy with 7 daily days plus weekly and monthly rollups. If you relied on the older 30-day daily retention behavior, update the backup settings after upgrading.
- This release includes the upstream sync merge and the related shared contract updates. Let normal startup apply migrations automatically, or run `pnpm db:migrate` if you manage upgrades manually.
