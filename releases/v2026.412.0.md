# v2026.412.0

> Released: 2026-04-12

## Highlights

- **Codex fast mode and broader local-model coverage** — Codex Local now supports a dedicated fast mode, and the updated model/config surfaces improve setup for Codex, Qwen, Gemini, Cursor, OpenCode, and related local adapters. ([#47](https://github.com/penclipai/paperclip-cn/pull/47), @penclipai)
- **Inbox and issue workflows feel faster to navigate** — The inbox, issue detail, quicklook, and reference handling paths now support better issue search fallback, smoother quicklook routing, and more polished thread/document interactions. ([#47](https://github.com/penclipai/paperclip-cn/pull/47), @cryppadotta, @penclipai)
- **Network binding and authenticated deployment setup are clearer** — The CLI and server now expose safer bind presets for loopback, LAN, and tailnet-style private deployments, making first-run setup and access diagnostics more predictable. ([#47](https://github.com/penclipai/paperclip-cn/pull/47), @cryppadotta, @penclipai)

## Improvements

- **Paperclip CN stayed aligned with upstream while preserving zh-CN UX** — The upstream sync kept configuration, activity, inbox, issue detail, and skill-management surfaces localized in `zh-CN`, including new Codex and configuration copy introduced upstream. ([#47](https://github.com/penclipai/paperclip-cn/pull/47), @penclipai)
- **Dev and worktree startup flows are more reliable** — Dev runner startup now avoids extra migration-status races, worktree env bootstrapping is tighter, and local verification remains stable on Windows even under the fork’s heavier test matrix. ([#47](https://github.com/penclipai/paperclip-cn/pull/47), @penclipai)
- **Plugin and adapter packaging is more resilient** — Shared adapter/server utilities, published package wiring, and generated workspace links were tightened so local and packaged environments stay closer together. ([#47](https://github.com/penclipai/paperclip-cn/pull/47), @officialasishkumar, @penclipai)

## Fixes

- **Release smoke now validates the real Docker host path** — The release smoke harness now boots authenticated canary installs with an explicit non-loopback bind preset, so published Docker validation probes the same host-mapped path that real operators use. ([#48](https://github.com/penclipai/paperclip-cn/pull/48), @penclipai)
- **Configuration and issue detail pages no longer leak English fallback copy in zh-CN** — The fork-specific localization pass restored translated configuration labels, parent-issue fallback text, activity verbs, and skill-source chrome that had regressed during the upstream sync. ([#47](https://github.com/penclipai/paperclip-cn/pull/47), @penclipai)
- **Full-suite verification is greener across local and CI environments** — The release train fixes several test and runtime regressions, including swipe/archive rendering in jsdom, workspace-runtime cleanup expectations, and Windows-specific worktree/db contention paths. ([#47](https://github.com/penclipai/paperclip-cn/pull/47), @penclipai)

## Upgrade Guide

- Existing `penclip` installs can upgrade in place; there is no package rename or new onboarding flow for this stable release.
- If you run Paperclip CN in private authenticated environments, prefer the newer bind presets (`loopback`, `lan`, or `tailnet`) when reconfiguring access, rather than relying on ad-hoc host overrides.
- This release includes the full upstream sync and the related schema/runtime updates. Let normal startup apply migrations automatically, or run `pnpm db:migrate` if you manage upgrades manually.
