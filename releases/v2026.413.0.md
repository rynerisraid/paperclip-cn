# v2026.413.0

> Released: 2026-04-13

## Highlights

- **Scoped issue wakes are more reliable end to end** — Issue-bound heartbeats now auto-check out the current task more safely, handle checkout conflicts gracefully, and keep signoff flows moving even when a wake already claimed the issue. ([#3538](https://github.com/paperclipai/paperclip/pull/3538), [#52](https://github.com/penclipai/paperclip-cn/pull/52), @cryppadotta, @penclipai)
- **Execution workspaces reuse existing linked worktrees instead of fighting them** — Paperclip can now detect already-checked-out branches, reuse the matching linked worktree, and avoid deleting reused worktrees during cleanup. ([#3538](https://github.com/paperclipai/paperclip/pull/3538), [#52](https://github.com/penclipai/paperclip-cn/pull/52), @cryppadotta, @penclipai)
- **Stable releases now ship a downloadable Windows desktop installer** — The GitHub stable release workflow now builds the Electron Windows installer, versions it with the stable release number, and uploads it to the same GitHub Release as the npm publish. ([#51](https://github.com/penclipai/paperclip-cn/pull/51), @penclipai)

## Improvements

- **Desktop runtime paths and Windows wrapper behavior are more predictable** — The desktop app keeps its app-data directory on the `penclip` slug, the dev runner follows the corrected path chain, and Windows `.cmd` wrapper argument forwarding is more robust. ([#50](https://github.com/penclipai/paperclip-cn/pull/50), @penclipai)
- **Issue detail stays cleaner after runs finish** — The board now drops stale active-run state once an issue is closed and unlocked, reducing misleading live-run chrome on completed work. ([#3538](https://github.com/paperclipai/paperclip/pull/3538), [#52](https://github.com/penclipai/paperclip-cn/pull/52), @cryppadotta, @penclipai)
- **Gemini adapter prompt wiring stays closer to upstream** — The CN fork trims Gemini-specific prompt drift and keeps the adapter closer to upstream while preserving the required package namespace boundary. ([#52](https://github.com/penclipai/paperclip-cn/pull/52), @penclipai)

## Fixes

- **zh-CN issue detail and onboarding copy no longer slip back to English during syncs** — Merge-touched issue detail toasts, breadcrumbs, and the CEO heartbeat checklist were re-localized so the touched product surfaces remain consistent in `zh-CN`. ([#52](https://github.com/penclipai/paperclip-cn/pull/52), @penclipai)
- **Runtime guidance no longer over-prescribes `curl --data-binary` payload handling** — The fork-specific runtime note was relaxed and then narrowed toward CLI-first guidance, removing an unnecessary hard requirement that was not part of upstream Gemini adapter behavior. ([#52](https://github.com/penclipai/paperclip-cn/pull/52), @penclipai)
- **Sync verification now covers the reused-worktree cleanup regression** — The release includes a regression test ensuring reused linked worktrees are not force-removed during teardown. ([#52](https://github.com/penclipai/paperclip-cn/pull/52), @penclipai)

## Upgrade Guide

- Existing `penclip` installs can upgrade in place; there is no package rename or onboarding reset in this release.
- If you use the Windows desktop app, you can now install from the GitHub Release asset directly instead of building the Electron package locally.
- If you run issue-scoped heartbeats or linked git worktrees, this release is recommended because it fixes both checkout conflict handling and reused-worktree cleanup safety.
