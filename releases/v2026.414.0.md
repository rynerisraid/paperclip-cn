# v2026.414.0

> Released: 2026-04-14

## Highlights

- **Issue-to-issue navigation feels faster and more reliable** — Issue detail links now prefetch more aggressively, reset scroll correctly during navigation, and tighten cache hydration so moving between related issues is smoother. ([#54](https://github.com/penclipai/paperclip-cn/pull/54), [#3542](https://github.com/paperclipai/paperclip/pull/3542), @cryppadotta, @penclipai)
- **Desktop windows no longer show the extra right-side scrollbar** — The Electron renderer now sizes its root layout to the real content area below the custom titlebar, which removes the slight vertical overflow that made the packaged shell look like it had two scrollbars. (@penclipai)

## Improvements

- **Desktop height handling is now centralized** — Electron preload publishes a shared available-height contract, the root UI shell consumes it, and desktop smoke now fails if the document/root layout overflows the Electron content area. (@penclipai)
- **The CN fork preserved touched-file localization during the upstream navigation sync** — The latest issue quicklook fallback copy remains translated in `zh-CN`, avoiding an English regression on the issue detail flow. ([#54](https://github.com/penclipai/paperclip-cn/pull/54), @penclipai)

## Fixes

- **Quicklook preview failures are now localized** — When an issue preview cannot load, the quicklook fallback message now respects the locale catalogs instead of surfacing hardcoded English. (@penclipai)
- **Desktop regression coverage is stronger for layout overflow** — The desktop smoke harness now asserts that the root page and scrolling element do not exceed the Electron content height on loaded pages. (@penclipai)

## Upgrade Guide

- Existing `penclip` installs can upgrade in place; there is no package rename or migration workflow change in this release.
- Desktop users do not need to change configuration. The fix is in the packaged Electron shell and renderer layout only.
