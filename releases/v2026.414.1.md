# v2026.414.1

> Released: 2026-04-14

## Highlights

- **Stable GitHub Releases now publish desktop downloads for every supported host** — The desktop release workflow now builds and attaches Windows x64, macOS x64, macOS arm64, and Linux x64 installers from the same stable release version, so users can download the right desktop package directly from one GitHub Release. ([#56](https://github.com/penclipai/paperclip-cn/pull/56), @penclipai)

## Improvements

- **Desktop packaging now shares one internal artifact contract** — Electron packaging, packaged smoke validation, and GitHub Release uploads now coordinate through `desktop-artifacts.json`, which removes the old Windows-only path assumptions and keeps desktop release automation aligned across platforms. ([#56](https://github.com/penclipai/paperclip-cn/pull/56), @penclipai)
- **Stable desktop assets are versioned from the released tag** — The release workflow now builds desktop artifacts from the stable tag that was just published, which keeps installer names and release uploads aligned with the exact released commit. ([#56](https://github.com/penclipai/paperclip-cn/pull/56), @penclipai)

## Fixes

- **Packaged desktop smoke is no longer tied to `win-unpacked` discovery** — The packaged smoke harness now reads the generated desktop artifact manifest, making release validation resilient to the new macOS and Linux packaging outputs. ([#56](https://github.com/penclipai/paperclip-cn/pull/56), @penclipai)

## Upgrade Guide

- Existing installs can upgrade in place; this release changes desktop packaging and release delivery only.
- Desktop artifacts remain unsigned in this phase, and macOS continues to ship as separate x64 and arm64 builds instead of a universal app.
