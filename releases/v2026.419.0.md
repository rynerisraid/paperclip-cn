# v2026.419.0

> Released: 2026-04-19

## Highlights

- **Multi-user access and invite flows** — Paperclip CN now carries the latest upstream authenticated user, company membership, invite landing, join request, profile, and access-management surfaces while preserving the fork's zh-CN defaults and company-boundary checks. ([#62](https://github.com/penclipai/paperclip-cn/pull/62), @penclipai)
- **Localized issue and invite UX after upstream sync** — Restored Chinese coverage for issue filters, creator and assignee controls, issue list empty/search states, project issue menus, invite landing states, and account/menu chrome introduced by the upstream sync. ([#63](https://github.com/penclipai/paperclip-cn/pull/63), @penclipai)
- **Language switching inside the account menu** — The main UI language selector now lives in the account menu below the theme toggle, while the onboarding dialog keeps its own local language switcher for first-run setup. ([#63](https://github.com/penclipai/paperclip-cn/pull/63), @penclipai)

## Improvements

- **Instance and project settings localization** — Instance General settings, project configuration, deployment/auth status badges, and project codebase controls now render with zh-CN copy instead of inherited English labels.
- **Direct locale-aware HTML serving** — Static `/index.html` requests now preserve the same locale injection behavior as the SPA fallback, including `Content-Language` and `Vary: Accept-Language`.
- **Production image tooling** — The production Docker image installs GitHub CLI from Debian's package repository and includes `openssh-client` plus `jq` for deployment environments that need native SSH and JSON tooling.
- **Claude model list update** — The Claude local adapter dropdown includes the latest Opus 4.7 option.
- **Routine list schedule data** — Routine list responses include trigger `cronExpression` and `timezone`, keeping API consumers and future UI list caches aligned with actual schedules.
- **Agent routine inbox continuity** — Agent-facing inbox-lite requests include routine execution issues so routine-driven pipelines can continue after delegation.

## Fixes

- **zh-CN merge drift** — Fixed several post-sync regressions where upstream issue, invite, column-picker, account-menu, and settings strings bypassed the locale catalogs.
- **PR and release verification** — Updated the language-switcher e2e test to follow the account-menu language entry and kept PR policy, e2e, verify, and Windows compatibility gates green after the UI relocation.
- **Desktop release artifacts** — Compressed and constrained desktop release artifacts so stable desktop assets remain shippable across supported platforms.
- **macOS and CI desktop packaging** — Kept macOS desktop builds unsigned-first and unblocked the desktop release workflow on current GitHub-hosted runners.
- **Windows path and worktree behavior** — Preserved Windows-safe path normalization and worktree provisioning semantics while absorbing upstream workspace/runtime changes.

## Upgrade Guide

This release includes the upstream multi-user access schema changes and follow-up join-request cleanup migration carried through PR #62. Deployments should run migrations automatically on startup before accepting traffic.

The migration set is intended to be additive, but deployments with unusual historical duplicate pending human join requests should review the join-request cleanup behavior before upgrade.

For Paperclip CN operators, no manual UI migration is required. Existing browser language preferences remain respected; the language switcher has moved from the sidebar footer into the account menu.
