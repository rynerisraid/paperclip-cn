# v2026.424.0

> Released: 2026-04-24

## Highlights

- **Local and SSH execution environments** — Paperclip can now define company environments, assign them at project and agent boundaries, lease local environments for runs, and execute supported local adapters through SSH-backed remote targets. ([#4297](https://github.com/paperclipai/paperclip/pull/4297), [#4358](https://github.com/paperclipai/paperclip/pull/4358), [#66](https://github.com/penclipai/paperclip-cn/pull/66), @devinfoley, @penclipai)
- **Issue subtree pause, cancel, restore, and deferred wakeups** — Operators can pause an issue subtree, cancel or restore descendant work, and allow human comments to defer agent wakeups until the subtree is resumed. ([#4332](https://github.com/paperclipai/paperclip/pull/4332), [#66](https://github.com/penclipai/paperclip-cn/pull/66), @cryppadotta, @penclipai)
- **Structured issue-thread interactions** — Issue threads now carry richer request/response cards, suggested-task handling, queued-comment recovery, and issue-reference context for more reliable agent collaboration. ([#4214](https://github.com/paperclipai/paperclip/pull/4214), [#4244](https://github.com/paperclipai/paperclip/pull/4244), @cryppadotta)
- **Plugin orchestration and safer adapter UI parsing** — Plugins gain orchestration host APIs, scoped plugin database support, and sandboxed parser execution so external adapter UIs can be loaded with a tighter trust boundary. ([#4114](https://github.com/paperclipai/paperclip/pull/4114), [#4225](https://github.com/paperclipai/paperclip/pull/4225), @cryppadotta)

## Upgrade Guide

This release includes database migrations `0058` through `0068`, adding plugin database namespaces, issue reference and interaction records, environment records and leases, issue tree holds, and agent default environment configuration. Deployments should run migrations automatically on startup before accepting traffic.

No manual configuration is required for existing local-only deployments. SSH environments are opt-in: create company environments and assign them to projects or agents only when remote execution is desired.

For Paperclip CN operators, no manual language migration is required. Existing zh-CN defaults and browser language preferences continue to apply; newly merged environment and issue-control surfaces have matching locale entries.
