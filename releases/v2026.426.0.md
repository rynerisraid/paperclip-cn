# v2026.426.0

> Released: 2026-04-26

## Highlights

- **Sandbox execution and provider plugins** — Paperclip can now model sandbox-backed execution through provider-neutral core APIs, with E2B available as the first external sandbox provider plugin. ([#4415](https://github.com/paperclipai/paperclip/pull/4415), [#4449](https://github.com/paperclipai/paperclip/pull/4449), [#4452](https://github.com/paperclipai/paperclip/pull/4452), @devinfoley)
- **Runtime recovery and live issue visibility** — Operators get stronger recovery behavior for liveness gaps, run continuations, stale output watchdogs, and issue graph visibility. ([#4419](https://github.com/paperclipai/paperclip/pull/4419), [#4446](https://github.com/paperclipai/paperclip/pull/4446), @cryppadotta)
- **CN fork upstream sync** — The CN fork is synced with upstream recovery, sandbox, plugin, and UI work while preserving `@penclipai/*` package scope, zh-CN localization, Windows compatibility, and external plugin boundaries. ([#68](https://github.com/penclipai/paperclip-cn/pull/68), @penclipai)

## Improvements

- **Plugin SDK and standalone release packaging** — Plugin packages, generated plugin surfaces, and standalone public package release packaging are more reliable for publishing and installation. ([#4494](https://github.com/paperclipai/paperclip/pull/4494), @devinfoley)
- **Issue and comment workflows** — Issue composer behavior, long document display, issue-thread review flows, markdown issue references, and escaped multiline issue or approval text were tightened for day-to-day operator work. ([#4381](https://github.com/paperclipai/paperclip/pull/4381), [#4382](https://github.com/paperclipai/paperclip/pull/4382), [#4420](https://github.com/paperclipai/paperclip/pull/4420), [#4444](https://github.com/paperclipai/paperclip/pull/4444), @cryppadotta)
- **Model refresh and skill governance** — Codex model refresh behavior, transient recovery, company skill detail loading, and create-agent governance assets are more robust. ([#4380](https://github.com/paperclipai/paperclip/pull/4380), [#4383](https://github.com/paperclipai/paperclip/pull/4383), [#4422](https://github.com/paperclipai/paperclip/pull/4422), @cryppadotta)
- **Test and maintenance stability** — Serialized route tests and local maintenance assets were stabilized, and the CN fork merge harness now runs more predictably on Windows. ([#4423](https://github.com/paperclipai/paperclip/pull/4423), [#4448](https://github.com/paperclipai/paperclip/pull/4448), [#68](https://github.com/penclipai/paperclip-cn/pull/68), @cryppadotta, @penclipai)

## Fixes

- **Watchdog access control** — Stale-run watchdog decisions now require board access before taking action. ([#4446](https://github.com/paperclipai/paperclip/pull/4446), @cryppadotta)
- **Retry cleanup** — Stale retries are cancelled when issue ownership changes, avoiding work resuming against the wrong assignee. ([#4445](https://github.com/paperclipai/paperclip/pull/4445), @cryppadotta)
- **Markdown rendering** — External links and issue-reference punctuation render more cleanly in markdown-heavy surfaces. ([#4447](https://github.com/paperclipai/paperclip/pull/4447), [#4382](https://github.com/paperclipai/paperclip/pull/4382), @cryppadotta)
- **Release package coverage** — Standalone public packages are included correctly during release packaging. ([#4494](https://github.com/paperclipai/paperclip/pull/4494), @devinfoley)

## Upgrade Guide

This release includes database migrations `0069_liveness_recovery_dedupe` and `0070_active_run_output_watchdog`. Deployments should run migrations through the normal startup flow before accepting traffic.

No manual configuration is required for existing local-only deployments. Sandbox providers are opt-in: install and configure provider plugins before assigning sandbox-backed execution environments.

For Paperclip CN operators, no manual language migration is required. Existing zh-CN defaults and browser language preferences continue to apply, and the fork-specific package and plugin boundaries remain preserved.
