# v2026.505.0

> Released: 2026-05-05

## Highlights

- **Remote execution environments** - SSH, sandbox, and local execution targets are now managed through a dedicated company environment settings surface, with adapter support matrices, connection probes, and provider-backed sandbox configuration.
- **Sandbox callback bridge** - Remote and sandboxed runs can call back into Paperclip through a managed bridge, enabling safer environment-aware run execution and plugin provider flows.
- **ACPX local adapter** - ACPX joins the built-in adapter catalog under the `@penclipai/adapter-acpx-local` package, with CLI event formatting, configuration schema support, UI parsing, test coverage, and CI release enrollment.
- **Routine and issue recovery operations** - Operators get routine run history, manual routine run visibility, issue monitor liveness controls, stale heartbeat retries, and configurable automatic recovery task creation for dependency-chain incidents.
- **Cloud tenant issue identifiers** - Issue routes, live run polling, activity, costs, and UI issue links now accept tenant-style identifiers with alphanumeric prefixes such as `PC1897-1`.
- **Paperclip CN localization preservation** - Merge-touched routine, company settings, access, environment, plugin, and instance settings pages have zh-CN chrome restored after the upstream sync.

## Improvements

- **Workspace and sidebar workflow polish** - Workspace switching now lives in the sidebar, nested issue surfaces are easier to scan, sub-issues can render as ordered workflow checklists, and sidebar agents expose pause/resume actions.
- **Runtime command specifications** - Adapters can declare runtime command specs for remote provisioning, model probes are validated against execution targets, OpenCode model selection is explicit and local-aware, and inherited host shell environment is stripped from SSH remote execution.
- **Release and package safety** - Registry verification, release manifest checks, dry-run package bootstrap validation, and dist-tag safeguards make canary and stable publishing harder to misconfigure.
- **Database backup coverage** - Non-system schemas are included in backup handling, with instance-level retention controls for automatic database backups.
- **Deployment documentation** - The release includes an AWS ECS Fargate runbook, deployment examples, Docker timeout improvements, and updated contributor/developer skills.

## Fixes

- Fixed remote workspace environment shaping, SSH callback URL selection for LAN/private networks, stale remote session resumes, plugin startup races, orphaned leases, and workspace sync behavior.
- Fixed disappearing issue comments, external issue URL rewriting in markdown, flaky heartbeat recovery teardown, stale queued heartbeat cancellation, and stale company skill refreshes.
- Fixed onboarding and CI reliability around live adapter probes, Docker build timeouts, PR path-diff policy, lockfile synchronization, and serialized server suites.
- Fixed release bootstrap for the newly published ACPX package so future CI releases include it instead of silently skipping it.

## Upgrade Guide

- Run the normal database migration path before opening the board. This release includes new migrations through `0075_cultured_sebastian_shaw`.
- If you run remote or sandbox execution, review the new company environment settings and confirm SSH/sandbox provider configuration before assigning environments to projects or agents.
- If you maintain npm publishing for the CN fork, keep `@penclipai/adapter-acpx-local` trusted publishing or token permissions aligned with the `penclipai/paperclip-cn` release workflow.
