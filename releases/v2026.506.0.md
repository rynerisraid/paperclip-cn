# v2026.506.0

> Released: 2026-05-06

## Highlights

- **Expanded plugin host capabilities** - Plugins can now use a broader host API surface, including scoped API routes, local folder access, managed agents, managed routines, richer UI helpers, and orchestration-facing SDK utilities. ([#5205](https://github.com/paperclipai/paperclip/pull/5205))
- **Routine revision history and restore** - Routine edits are now tracked through revision history so operators can inspect earlier routine versions and restore prior definitions when needed. ([#5285](https://github.com/paperclipai/paperclip/pull/5285))
- **More reliable local and sandboxed adapters** - Adapter execution now has better environment isolation, sandbox command probing, stdin delivery, and per-adapter install command behavior across local, SSH, and sandbox targets. ([#5274](https://github.com/paperclipai/paperclip/pull/5274), [#5275](https://github.com/paperclipai/paperclip/pull/5275), [#5277](https://github.com/paperclipai/paperclip/pull/5277), [#5278](https://github.com/paperclipai/paperclip/pull/5278), [#5279](https://github.com/paperclipai/paperclip/pull/5279), [#5280](https://github.com/paperclipai/paperclip/pull/5280))

## Improvements

- **Gemini CLI compatibility** - Gemini stream-json parsing now handles the v0.38 wire format consistently across server parsing, CLI formatting, and UI transcript rendering. ([#5273](https://github.com/paperclipai/paperclip/pull/5273))
- **Codex API-key authentication** - Codex adapter setup now writes API-key-mode auth data compatible with Codex CLI 0.122 and newer when using `OPENAI_API_KEY`. ([#5276](https://github.com/paperclipai/paperclip/pull/5276))
- **Paperclip CN localization preservation** - Merge-touched routine, plugin, project, onboarding, and adapter-management UI surfaces keep zh-CN coverage after the upstream sync. ([#71](https://github.com/penclipai/paperclip-cn/pull/71))
- **Windows external adapter installs** - External adapter package installs now normalize Windows home paths, use junctions for directory links, and avoid Unix-only package scripts in published workspace packages. ([#71](https://github.com/penclipai/paperclip-cn/pull/71))
- **Desktop packaging readiness** - The Electron package flow was rechecked after the upstream plugin/runtime changes, including the larger bundled runtime introduced by built-in ACPX support. ([#71](https://github.com/penclipai/paperclip-cn/pull/71))

## Fixes

- Fixed development startup on Windows when reserved local Postgres ports are occupied.
- Fixed packaged external adapter install resolution so npm-installed adapters such as Hermes can be found from the packaged runtime.
- Fixed company creation and built-in/external adapter visibility regressions introduced during the upstream merge.
- Fixed lockfile metadata drift surfaced by the GitHub policy runner.

## Upgrade Guide

- Database migrations run through the normal Paperclip startup path. This release adds routine revision and plugin-managed-resource tables; no manual migration step is expected for standard deployments.
- Desktop installers are larger than previous stable builds because the bundled `acpx_local` runtime includes native Claude and Codex ACP binaries. This is expected for this release, not a packaging cache leak.
- Windows users installing external adapters should prefer the adapter manager or npm package flow; directory links are now materialized as junctions where Windows cannot create ordinary symlinks reliably.
