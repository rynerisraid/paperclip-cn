# v2026.507.0

> Released: 2026-05-07

## Highlights

- **Company-wide search** - Operators can now search across a company from a dedicated board search page, with Command-K handoff, highlighted matches, recent searches, company-scoped backend contracts, rate limiting, and document/title matching. ([#5293](https://github.com/paperclipai/paperclip/pull/5293))
- **Planning mode for issues** - Issues can now be created and carried as `standard` or `planning` work, with shared contracts across the database, API validators, plugin protocol, heartbeat context, issue rows, and the issue detail composer. ([#5353](https://github.com/paperclipai/paperclip/pull/5353))
- **Safer remote and sandbox execution** - SSH and sandbox targets now have tighter environment shaping, a broader documented callback surface, serialized callback bridge writes, stdin-safe SSH uploads, and longer Gemini remote hello probes. ([#5322](https://github.com/paperclipai/paperclip/pull/5322), [#5324](https://github.com/paperclipai/paperclip/pull/5324), [#5325](https://github.com/paperclipai/paperclip/pull/5325), [#5326](https://github.com/paperclipai/paperclip/pull/5326))

## Improvements

- **Issue audit trail clarity** - Issue threads now show workspace-change activity, stale disposition notices, and recovery handoff notices in the operator timeline with focused rendering and route coverage. ([#5289](https://github.com/paperclipai/paperclip/pull/5289), [#5356](https://github.com/paperclipai/paperclip/pull/5356), [#74](https://github.com/penclipai/paperclip-cn/pull/74))
- **Operator workflow polish** - Sidebar and issue controls now include improved issue property timestamps, company ordering controls, navigation/status layout polish, and better repeated-board ergonomics. ([#5291](https://github.com/paperclipai/paperclip/pull/5291), [#5355](https://github.com/paperclipai/paperclip/pull/5355))
- **Control-plane safety** - Manual heartbeat invokes preserve their operator-supplied scope, issue identifiers support cloud-style alphanumeric prefixes, and agent-authored review transitions are guarded when no real review path exists. ([#5323](https://github.com/paperclipai/paperclip/pull/5323), [#5292](https://github.com/paperclipai/paperclip/pull/5292))
- **ACPX adapter configuration** - ACPX local adapter setup has improved configuration schema and UI parsing support for model and execution settings. ([#5290](https://github.com/paperclipai/paperclip/pull/5290))
- **Paperclip CN sync preservation** - The fork absorbed the latest upstream search, planning, recovery, issue-thread, and sidebar work while preserving zh-CN locale coverage, Windows compatibility checks, Electron packaging validation, `@penclipai/*` package boundaries, and external-only Hermes/Droid adapter loading. ([#73](https://github.com/penclipai/paperclip-cn/pull/73), [#74](https://github.com/penclipai/paperclip-cn/pull/74))

## Fixes

- Fixed sandbox callback bridge races that could corrupt concurrent heartbeat response files or entrypoint uploads.
- Fixed remote execution environment leakage where host identity variables could bleed into SSH or sandbox targets.
- Fixed manual heartbeat invoke scope loss from the live-run route.
- Fixed planning-mode e2e coverage under the CN fork's default zh-CN locale.
- Fixed full-suite Vitest stability for side-effect-heavy server suites by isolating known serialized runs in the stable runner.
- Fixed merge-touched issue/system/sidebar UI strings so English and zh-CN locale catalogs remain in sync.

## Upgrade Guide

- Run the normal Paperclip startup/migration path before opening the board. This release includes search indexes, fuzzy search support, and issue work-mode migrations through `0081_optimal_dormammu`; no manual migration step is expected for standard deployments.
- Planning-mode issue data defaults missing values to `standard`, so existing issues and integrations should continue to work without payload changes.
- If you run remote or sandbox execution, review configured target environments after upgrade. The release intentionally strips host-shaped identity variables at remote boundaries.
- Paperclip CN desktop packaging was locally rechecked with the Windows x64 `pack` flow; the stable release workflow also builds the full unsigned desktop asset matrix.
