# v2026.508.0

> Released: 2026-05-08

## Highlights

- **Recover stalled work with clearer operator controls** - Issue details now include retry-now actions, scheduled retry state, monitor activity, retry cards, and ledger updates so operators can see why work is waiting and nudge it forward without losing context. ([#5426](https://github.com/paperclipai/paperclip/pull/5426), [#5427](https://github.com/paperclipai/paperclip/pull/5427))
- **Assigned backlog safeguards** - The control plane now detects assigned-backlog stalls more directly and surfaces safeguards around work that is blocked because the assigned agent has not picked it up. ([#5428](https://github.com/paperclipai/paperclip/pull/5428))
- **More resilient remote execution** - Remote and sandbox runtime paths now preserve and restore workspace changes more carefully, including restore-merge handling, safer SSH helpers, stronger runtime probes, and Cursor sandbox resolution fixes. ([#5444](https://github.com/paperclipai/paperclip/pull/5444), [#5445](https://github.com/paperclipai/paperclip/pull/5445), [#5446](https://github.com/paperclipai/paperclip/pull/5446))
- **Paperclip CN project and language polish** - New project setup accepts general HTTP(S) Git repository URLs instead of GitHub-only links, while the account menu uses the compact inline language switcher and keeps zh-CN/English switching intact. ([#77](https://github.com/penclipai/paperclip-cn/pull/77))

## Improvements

- **Issue workflow readability** - Issue rows, properties, blocked notices, monitor activity, scheduled retry cards, status badges, and related inbox states have broader zh-CN coverage while preserving upstream behavior.
- **Sidebar and navigation polish** - The sidebar search icon routes directly to search, and account-menu language switching now reuses the shared `LanguageSwitcher` logic without the old large explanatory block. ([#5440](https://github.com/paperclipai/paperclip/pull/5440), [#77](https://github.com/penclipai/paperclip-cn/pull/77))
- **Adapter runtime coverage** - Adapter utility tests now cover command-managed runtimes, sandbox callback behavior, SSH fixtures, server helpers, and workspace restore merge behavior across the updated runtime surfaces.
- **CN fork boundary preservation** - The sync kept `@penclipai/*` package boundaries, default zh-CN locale behavior, Windows-sensitive test coverage, and external adapter boundaries while absorbing upstream issue and runtime changes. ([#77](https://github.com/penclipai/paperclip-cn/pull/77))

## Fixes

- Fixed merge-touched UI tests that could suspend on i18n initialization by using non-suspense translation hooks in targeted components.
- Fixed account-menu language-switcher E2E coverage so CI tracks the compact inline control instead of the removed `Switch language` popover title.
- Fixed fork-local plugin worker tests to import `@penclipai/shared` and `@penclipai/plugin-sdk` instead of stale upstream package names.
- Fixed a Windows/full-suite-sensitive heartbeat dependency scheduling test by giving the local adapter wait enough time under the full Vitest run.
- Fixed missing zh-CN and English locale keys for sidebar agent pause/resume toasts and the new issue retry/backlog UI.

## Upgrade Guide

- No manual database migration step is expected beyond the normal Paperclip startup path. This release does not add new database migration files.
- If you rely on remote or sandbox execution targets, keep the normal workspace backup/restore expectations in place; this release makes restore behavior more careful but does not require new configuration.
- For Paperclip CN operators, existing GitHub project URLs continue to work. New project forms now also accept other `http://` and `https://` Git repository URLs; SSH-style remotes are still intentionally out of scope for this release.
