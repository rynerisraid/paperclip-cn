# v2026.508.1

> Released: 2026-05-08

## Highlights

- **Windows onboarding recovery** -- `penclip onboard --yes` now recovers from interrupted embedded PostgreSQL initialization when the data directory contains `PG_VERSION` but is missing `global/pg_control`. This removes the need to manually delete the half-created local database directory before retrying setup. ([#81](https://github.com/penclipai/paperclip-cn/pull/81), @penclipai)

## Fixes

- Reused the existing embedded PostgreSQL incomplete-cluster reset guard in the main server startup path before deciding whether `initdb` should run.
- Added regression coverage for the half-initialized data-directory shape seen during Windows onboarding failures.
